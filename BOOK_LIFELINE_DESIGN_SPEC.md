# Book Lifeline Demo — 完整设计开发文档

> 本文档供 Code Agent 开发使用。包含所有页面的设计规范、交互逻辑、设计原则和开发约束。
> **严重警告：开发过程中不得丢失或篡改原设计内容。所有 UI 元素、文案、颜色、字号、间距必须严格按本文档实现。**

---

## 一、产品概述

**产品名称**：Book Lifeline（书脉）
**核心理念**：AI 找回用户曾经在书里留下的自己。不是 AI 推荐书，不是 AI 生成精美卡片，而是 AI 从用户过去留下的读书摘要、日记和书脉输入中，找到与当前感受语义相关的旧痕迹。
**目标用户**：热爱阅读、有记笔记习惯的用户
**演示场景**：TRAE AI 创意比赛评审演示

---

## 二、设计原则（不可违反）

1. **Apple HIG 风格**：Clarity（清晰）、Deference（内容优先于 UI 控件）、Depth（层次感）
2. **零外部依赖**：不使用任何 CDN、外部字体、图标库。所有图标为内联 SVG，字体使用系统字体栈
3. **自包含 HTML**：每个页面是一个独立的 HTML 文件，内联 CSS 和 JS
4. **中文文件名**：页面文件使用中文命名
5. **信息克制**：页面标题不是必需的。如果页面内容本身已经能传达主题，就不加标题（Deference 原则）
6. **不展示标签给用户**：标签容易造成误解，只在最终结果中统一呈现，不在中间过程展示
7. **不使用提示性 badge 展示普通信息**：条目数量是普通信息，用普通文本展示，不用圆形数字 badge
8. **NEW 标志可以保留**：用于提示有新内容，影响排序

### 3.3 容器规范
- 最大宽度：640px（来源页和旧回声推荐页为 820px，书脉总览页为 960px）
- 居中显示：`margin: 0 auto`
- 内容区底部留白：`padding-bottom: 72px`（为底部 Tab Bar 让出空间）

### 3.4 底部 Tab Bar 规范（所有页面必须包含）
- 高度：60px
- 固定在底部：`position: fixed; bottom: 0; left: 0; right: 0`
- 毛玻璃效果：`background: rgba(255,255,255,0.85); backdrop-filter: blur(20px); -webkit-backdrop-filter: blur(20px)`
- 上边框：`border-top: 0.5px solid #E5E5EA`
- 内边距：`padding-top: 8px; padding-bottom: env(safe-area-inset-bottom, 0px)`
- 三个 Tab 按顺序：**记录** | **书脉** | **来源**
- Tab 链接目标：
  - 记录 → `输入页.html`
  - 书脉 → `书脉总览页.html`
  - 来源 → `来源页.html`
- 激活态样式：图标背景 `#E8F2FF`（圆形），图标和文字均为 `#007AFF`，文字 `font-weight: 600`
- 非激活态：图标和文字均为 `#AEAEB2`，无背景
- 所有图标为内联 SVG，24x24 viewBox，`stroke-width: 1.5`，`stroke-linecap: round; stroke-linejoin: round`，无 fill（使用 currentColor）
- **AI 分析中页隐藏底部 Tab Bar**（`display: none`）

### 3.5 Tab 图标 SVG 路径
- **记录**（笔记本）：`M4 4h12a2 2 0 0 1 2 2v14H6a2 2 0 0 1-2-2V4z` + `M16 2H8a2 2 0 0 0-2 2v2h12V4a2 2 0 0 0-2-2z` + 两条横线
- **书脉**（连接节点）：三个圆 + 连线
- **来源**（档案盒）：`M3 7h18v12a2 2 0 0 1-2 2H5a2 2 0 0 1-2-2V7z` + `M8 7V5a2 2 0 0 1 2-2h4a2 2 0 0 1 2 2v2` + 两个圆点

### 3.6 卡片规范
- 背景：`var(--fill-primary)` 即 `#FFFFFF`
- 边框：`1px solid var(--separator)` 即 `1px solid #E5E5EA`
- 圆角：`var(--radius)` 即 `12px`
- 内边距：通常 `20px`（内容密集的页面可用 `24px`）
- 阴影：默认无。hover 时 `box-shadow: 0 4px 16px rgba(0,0,0,0.06)`

## 四、页面流程与交互逻辑

### 4.1 完整流程图
```
输入页
  │ [btn-start] "开始回捞"
  ▼
节点确认页
  │ [btn-back-edit] "返回修改" → 返回输入页
  │ [btn-confirm-echoes] "确认，开始回捞"
  ▼
AI 分析中页（过渡页，底部 Tab 隐藏）
  │ [auto-next] 自动跳转（约 3 秒）
  │ [返回箭头] → 返回节点确认页
  ▼
  ├── 找到旧痕迹 → 旧回声推荐页
  │     │ [btn-confirm-lifeline] "确认关系，进入书脉"
  │     ▼
  │   书脉回看页（过程留痕总结页）
  │     │ [btn-continue-input] "继续录入新想法" → 输入页
  │     │ [btn-view-overview] "查看完整书脉" → 书脉总览页
  │     │ [返回箭头] → 旧回声推荐页
  │     ▼
  │
  └── 未找到旧痕迹 → 新节点确认页
        │ [btn-go-overview] "进入书脉" → 书脉总览页
        │ [btn-continue-input] "继续记录" → 输入页
        │ [返回箭头] → 输入页
        ▼

书脉总览页（Tab 入口：书脉）
  │ [book-detail-1] 点击书籍 → 书籍详情页
  │ [btn-new-input-bottom] "输入新想法" → 输入页
  ▼
书籍详情页
  │ [btn-back-overview] 返回 → 书脉总览页
  ▼

来源页（Tab 入口：来源）
```

### 4.2 分支逻辑说明
- AI 分析后有两种结果：找到旧痕迹（走旧回声推荐页）或未找到（走新节点确认页）
- 旧回声推荐页中，用户可以逐条确认/放弃。全部放弃等同于未找到，也应进入新节点确认页
- 新节点确认页有两种变体：与书相关（显示书籍信息）和与生活相关（不显示书籍信息）

## 九、需要强调的重点（评委关注点）

1. **AI 找回的是"语义关联"不是"关键词匹配"**：旧回声推荐页的"为什么与此刻相关"框是证明这一点的核心。解释文字必须详细、具体、有说服力
2. **"不重写用户原文"**：节点确认页的原始输入可编辑但 AI 整理版独立展示，两者是对照关系而非替代关系
3. **三种来源的清晰区分**：读书摘要（绿）、日记（橙）、书脉输入（蓝），贯穿来源页和所有引用场景
4. **书卡像真正的书**：用真实封面、克制的信息展示，不是数据卡片
5. **流程自然流畅**：输入→确认→分析→回声→书脉回看，每一步都有明确的用户决策点
6. **空状态不是失败**：新节点确认页传递"独立节点也有价值"的信息
